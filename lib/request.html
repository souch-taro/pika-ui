# Taro-Request 请求库

  1. 默认的请求头: 'content-type': 'application/x-www-form-urlencoded'

# 安装和快速上手

安装:

yarn add @souche-taro/request

使用:

import http from "@souche-taro/request"

http.request({
    method:'get',
    url: 'http://www.abc.com/test',
    data: {
        a: 1,
        b: 2
    }
})

# 初始化实例

let instance = http.create({
    baseURL: 'http://www.abc.com'
});

let res = await instance.get('/test', {a:1,b:2})
// 等价于
let res = await instance.request({
    method:'GET',
    url:'/test',
    data: {
        a:1,
        b:2
    }
})

# 别名

http.get(url, data?, config?) 
// => http.request({url,data,method:'GET',...config})
http.post(url, data?, config?) 
// => http.request({url,data,method:'POST',...config})
http.put(url, data?, config?) 
// => http.request({url,data,method:'PUT',...config})
http.delete(url, data?, config?) 
// => http.request({url,data,method:'DELETE',...config})

# 重置配置

// init config
let instance = http.create(config);

// new config
instance.loadConfig(newConfig);

# API 说明

基础参数:

参数名称 类型 必须 默认 说明
url string 否 接口路径
data U 否 请求的参数
method "OPTIONS" | "GET" | "HEAD" | "POST" | "PUT" | "DELETE" | "TRACE" | "CONNECT" 否 "GET" 请求方法

扩展选项:

V0.1.0 老版本遵循的是Taro官方的参数

参数名称 类型 必须 默认 说明
baseURL string 否 公共的基础URL
soucheMode boolean 否 true 默认以搜车约定的格式返回数据, false可以返回完整response
quiet boolean 否 false 沉默错误提示
transformRequest function/function[] 否 接受函数或者函数数组, 用于改变发出请求前改变请求值
transformResponse function/function[] 否 接受函数或者函数数组, 用于改变获得请求后改变返回值
errTipTime number 否 2000 错误提示持续时间

V1.x.x 新版本遵循axios的配置规则, 下面是在此基础上新增的规则

参数名称 类型 必须 默认 说明
soucheMode boolean 否 true 默认以搜车约定的格式返回数据, false可以返回完整response
windowSize number 否 6 一次性请求最大接口数, 目前未区分域名
cache 缓存模式 否 'no-store' 缓存模式,详情见下方表格
cacheTime 缓存时间 否 180000 缓存的时间

cache参数说明

目前缓存是以请求path为标识的, 因此暂时不适用于分页数据

参数名称 说明
no-store 不走缓存,默认状态
cache 有缓存先走缓存,没有再请求
reload 有缓存先走缓存,然后再请求,返回数据,再次执行回调,如果没有缓存则走请求

# 其他说明

# data 参数说明

最终发送给服务器的数据是 String 类型，如果传入的 data 不是 String 类型，会被转换成 String 。转换规则如下：

对于 GET 方法的数据，会将数据转换成 query string（encodeURIComponent(k)=encodeURIComponent(v)&encodeURIComponent(k)=encodeURIComponent(v)...） 对于 POST 方法且 header['content-type'] 为 application/json 的数据，会对数据进行 JSON 序列化 对于 POST 方法且 header['content-type'] 为 application/x-www-form-urlencoded 的数据，会将数据转换成 query string （encodeURIComponent(k)=encodeURIComponent(v)&encodeURIComponent(k)=encodeURIComponent(v)...）
